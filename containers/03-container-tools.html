Container tools

A common concern when adopting a new technology is the right tools that should be used in order to leverage the best and efficient way of working to get better results, this is why it should be used tools that are aligned and compliant with OCI.

One of the most important changes that has been introduced from Red Hat in containers are tools that do not require root superpowers and it does not depend on a linux daemon, such as Docker, making it more flexible and reachable to a wide range of audience. Buildah, Podman and Skopeo are basic tools that are pretty handy to create, manage and distribute containers and container images aligned to be OCI compatible.

Buildah and Podman, the basis

Let’s take a look at to Buildah and Podman tools in deeper:

Buildah: its name comes from the word ‘builder’ pronounced with a Boston accent, a funny story that begins with a simple conversation, as Daniel Wash Senior Distinguished Engineer at Red Hat states in the article Introducing Buildah 1.0 “why can’t we create a simple tool to build them without running a container daemon? The engineers asked me what to call it and I responded: “just call it builder.” The engineers heard my Boston accent and Buildah was born”.

Buildah is basically a command line tool that is design to create and build container images. Despite being a tool designed to be OCI compatible, it could create images from a traditional Dockerfile too, which give flexibility when needed. Speaking of flexibility, Buildah is a perfect tool to be used in scripting and pipelines, actually it comes in default modules in kubernetes pipelines tools like tekton, which is pretty handy when lifecycle automation of a kubernetes application is needed.

Podman: is a command line tool that its name does not come from a funny story like buildah, but more like most of the tools, is an abbreviation of ‘Pod Manager’, and as its name points is designed to manage and run containers, images and volumes. The podman’s core is based on a golang library called libpod, which is in charge of all actions that the tool could perform. .

Podman project also has a child project called Podman Desktop, which allows the user to leverage all podman bondages from the desktop. In version v0.14 has introduced kind support to install kubernetes clusters for testing purposes. More details could be found on the project website.

container-tools is a group of packages in Fedora/Red Hat like distributions that includes the necessary packages to manage containers, such as: Buildah, Podman, Skopeo, Container-selinux, Containernetworking-plugins, Fuse-overlayfs, slirp4netns. Install as follows:

sudo dnf module install container-tools -y
