Open Container Initiative

Due to the fact that there are a lot of companies that started to use container technology and there was not a common standard to use it, a large Open Source community started to get around the idea of establishing some rules to standardize the management of this technology, getting under way the Open Container Initiative.

What is the Open Container Initiative

Letâ€™s start talking about the Open Container Initiative (OCI), which is an open project built from the community and sponsored by the Linux Foundation to define guidelines for container technology, which will state the requirements about the necessary features in the designing of containers formats and runtimes, to be followed by the industry.

OCI specifications

OCI has named these guidelines as specifications and currently has been developed the three specs below:

  • Distribution specification: a.k.a. distribution-spec, which is in charge of defining the requirements used to ease the distribution of agnostic content types in a standardized way through an API protocol.

  • Image specification: a.k.a. image-spec, the main purpose of this specification is to prepare images design to be compatible with tools that manage container images.The image-spec/config.md at mainimage spec should follow the below requirements:

    • Image Manifest.

      • Provides unique ID obtained from hashed configuration.

      • Multi-architectural images, grouped into a Main Manifest that can be referenced for platform specific versions.

      • Compatible with runtime-spec

    • Image index (Optional).

      • Acts as Metadata in a higher level to point specific image manifests

    • Filesystem layers. It defines how to build the filesystem that will be used by the image and how this will be populated, modified and tracked.

    • Configuration. It defines into a json document all properties needed to be applied in the container runtime.

  • Runtime specification: a.k.a. Runtime-spec, it defines the necessary metadata to handle how containers should be managed.
